MPPT chargers and temperature sensors (read side). A line physically shared with an inverter is configured here as a charger and separately in the inverters tab with the same port.

Feed-in units (write side). One sender per port; multiple identical units wired in parallel share a port via parallel units. max/min power are per single unit. Stage selects which power stage the group runs in — leave both unchecked to disable a unit without removing it.

Temperature alarms for eSmart3, EPever, Renogy, Morningstar and AGG temperature sensors. Each alarm fires when the temperature crosses its threshold and a command is set. Thresholds may be negative; an empty threshold or command disables that alarm. Device list is read from the saved configuration — add devices in the chargers tab first.

⚠ discharge_timer is disabled in zeroinput.conf — the timer file is saved but has no effect until enabled.

Format: YYYY-MM-DD HH:MM:SS <battery%> <inverter%> <energy_Wh> — use 0000-00-00 for today.

⚠ Restart zeroinput

This will stop and restart the zeroinput service via systemd.
All current inverter output stops for a few seconds.
Required after changes to chargers or inverters configuration.

If Volkszähler data stops updating after the restart, restart vzlogger as well using the button below.

⚠ Restart required

The chargers and inverters configuration is read once at startup and cannot be reloaded at runtime.

After saving, zeroinput must be restarted for changes to take effect:

sudo systemctl restart zeroinput
